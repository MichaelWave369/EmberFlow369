# EmberFlow369 Sprint v0.4-alpha — Receipt Export

## Sprint goal

Turn the v0.3 visual dashboard into a prototype evidence-export cockpit.

The key loop is now:

```text
mock hazard signal -> risk score -> action card -> receipt -> JSON export bundle
```

## What changed

- Added a browser-side receipt export utility in `apps/dashboard/src/exportBundle.ts`.
- Added an `Export receipts JSON` button to the dashboard.
- Added export bundle metadata, versioning, safety boundary text, action-card records, receipt records, source IDs, claim tags, and raw mock signals.
- Updated dashboard styling for the export panel.
- Bumped root and dashboard package versions to `0.4.0-alpha.0`.
- Updated README files to describe the export milestone.

## Export bundle contents

The export bundle includes:

- `bundleId`
- `generatedAt`
- `project`
- `version`
- `scope`
- `safetyBoundary`
- `scenarioCount`
- `highestRiskState`
- `actionCards`
- `receipts`
- `rawSignals`

## Claim discipline

The v0.4 export is still mock-only. It should be treated as a prototype review artifact, not an official incident record.

Current claim tags are inferred from the mock scenario and source IDs:

- `inferred`
- `estimated`
- `official-status-display`
- `witness`
- `sensor`

These tags are intentionally simple and should be replaced with stricter source classification when live feeds are introduced.

## Acceptance checks

- `npm run dashboard:build` should typecheck and build the dashboard.
- Dashboard should show an export panel below the processing loop.
- Clicking `Export receipts JSON` should download a JSON file named like:

```text
emberflow369-receipts-<timestamp>.json
```

- The downloaded file should include action cards, receipts, source IDs, claim tags, raw mock signals, and the safety boundary.

## Not included yet

- signed receipts
- persistent local storage
- server-side ledger writes
- live data feeds
- receipt diff viewer
- official incident archive export

## Next sprint candidate

v0.5 should connect the dashboard to the shared engine output package and add an in-app receipt viewer before export.
