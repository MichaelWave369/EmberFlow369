# Contributing to EmberFlow369

Thank you for helping build a safer, clearer, more accountable resilience layer.

## Project values

- Life safety first.
- Official instructions override local suggestions.
- Receipts over vibes.
- Clear uncertainty over false certainty.
- Privacy by default.
- Community reports need provenance.
- Corrections should be visible, not hidden.

## Good first contributions

- improve docs
- add mock data examples
- refine JSON schemas
- add tests for risk scoring
- research public data-source terms
- design action cards
- improve accessibility and plain-language wording

## Claim-safe contribution rule

Do not submit code or docs that imply EmberFlow369 is an official emergency alerting system. Local suggestions must be labeled as local suggestions.

## Data-source rule

Before adding real data ingestion, document:

- source name
- source owner
- source URL
- terms or license notes
- rate limits if known
- attribution requirements
- update cadence
- stale-data behavior

## Pull request checklist

- [ ] The change preserves official-vs-local boundaries.
- [ ] New claims include source and confidence language.
- [ ] User-facing safety wording is plain and non-alarming.
- [ ] Sensitive data is not exposed.
- [ ] Schemas or examples are updated if needed.
- [ ] Tests or validation notes are included where practical.

## Tone

This project should be calm, useful, and grounded. During disaster stress, people need clarity, not hype.
