# Sprint v0.5-alpha — GitHub Pages Dashboard

## Goal

Turn the EmberFlow369 React dashboard into a public GitHub Pages experience.

## Scope

This sprint adds deployment infrastructure only. It does not connect live incident feeds, official alerts, authenticated user data, or a server-side receipt ledger.

## Delivered

- Configured the Vite dashboard build for static hosting with relative asset paths.
- Added a GitHub Pages deployment workflow at `.github/workflows/pages.yml`.
- Added a root `dashboard:preview` script for local production-preview checks.
- Updated dashboard version text to `v0.5-alpha GitHub Pages`.
- Added `docs/GITHUB_PAGES.md` with setup and verification instructions.

## Deploy loop

```text
push to main -> build dashboard -> upload apps/dashboard/dist -> deploy to GitHub Pages
```

## Expected public URL

```text
https://michaelwave369.github.io/EmberFlow369/
```

## Acceptance checks

- `npm run dashboard:build` passes.
- The GitHub Pages workflow completes successfully.
- Repository Pages source is set to `GitHub Actions`.
- The live page loads the dashboard cards, safety boundary, and receipt export button.
- Exporting receipts JSON still works from the hosted page.

## Safety boundary

The hosted dashboard is still a public mock-data prototype. It must not be presented as an official emergency service, evacuation-order system, or disaster prediction guarantee.

## Next sprint candidate

v0.6 should add an in-app receipt viewer before download, so the hosted page can inspect action-card and receipt details without immediately exporting JSON.
