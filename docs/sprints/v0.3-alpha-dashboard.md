# EmberFlow369 v0.3-alpha Dashboard Sprint

## Purpose

Make the EmberFlow369 loop visible for the first time.

This sprint adds a tiny local dashboard that turns mock hazard signals into visual risk states, action cards, and receipt references.

## Loop demonstrated

```text
mock hazard signal -> risk score -> action card -> receipt reference -> visual cockpit
```

## What changed

- Added `apps/dashboard` Vite + React app.
- Added mock Northern California hazard scenarios.
- Added local dashboard risk scoring and official-status override behavior.
- Added action-card rendering.
- Added receipt identifiers and source tags in the UI.
- Added safety boundary banner.

## What is still mock-only

- Hazard signals are demo fixtures.
- Source IDs are mock tags, not live API receipts.
- Official statuses are simulated fixtures.
- No live CAL FIRE, NOAA, USGS, CDEC, NASA FIRMS, road, or county feeds are connected yet.

## Acceptance checks

A reviewer should be able to:

1. Run `npm install` from repo root.
2. Run `npm run dashboard` from repo root.
3. Open the local Vite URL.
4. See multiple hazard zones sorted by risk score.
5. See state labels, action cards, source tags, and receipt IDs.
6. See the safety boundary clearly displayed.

## Claim boundary

This dashboard is a preparedness and design demo only. It is not an official emergency alert, evacuation instruction, or life-safety authority.
